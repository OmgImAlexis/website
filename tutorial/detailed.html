This tutorial will help you install, run, and configure Caddy for your first time. (If you've used web servers before, skip to the Quick Start.) Although Caddy is very easy to use, it is still expected that you are already familiar with using your machine:

  • Extracting, moving, and renaming files
  • Managing users and file permissions
  • Using the terminal or command line
  • Configuring your firewall

With these prerequisites, this tutorial should be valid for all OS platforms. Some instructions may be OS-specific.

Quick Start

  1. Download Caddy and put it in your PATH.
  2. cd to the folder containing your website.
  3. Run caddy.

Caddy's default port is 2015, so open your browser to http://localhost:2015 to see it working. If you see a 404 error, then Caddy is working, but the root of your site is missing an index file.

Congratulations! You're up and running. Skip to the Configure section of the tutorial to do more.

Download

Download Caddy from the download page. You can get Caddy for nearly any OS and architecture, optionally configured with plugins for extra functionality.

You could also download the latest release from GitHub (without plugins).

Install

The file you downloaded is a compressed archive. You'll want to extract the Caddy binary (executable file).

Windows
Right-click the .zip file and choose "Extract All". Choose any folder to extract into, just don't lose track of it. You can delete it when we're done.
macOS
Double-click the .zip file to extract it, or run this command: unzip caddy*.zip caddy
Linux
Run this command: tar -xzf caddy*.tar.gz caddy

Next, we will move the Caddy binary into a folder where we can execute it easily.

Windows
Move the executable to any folder that's easy to get at. For example, C:\Caddy.
macOS and Linux
Any $PATH location will do: mv ./caddy /usr/local/bin If you get permission denied errors, you'll need to run as sudo.

Now Caddy is at our fingertips. Let's run it!

Run

By default, Caddy will use the current directory (the directory it is being executed from, not the folder where the binary lives) as the root of the site.

Using the terminal or command line, change into the folder where your site is:

cd path/to/my/site

And run Caddy:

Windows
Assuming you put the .exe file in C:\Caddy: C:\Caddy\caddy.exe
macOS and Linux
caddy

Load http://localhost:2015 in your browser. If you see a 404 error, then Caddy is working but your site is missing an index file.

You can quit Caddy by pressing Ctrl+C. It will terminate as gracefully as possible.

Configure

Your site is already fit for production, except it's on port 2015, not 80 (the standard HTTP port). Operating systems don't let programs bind to low ports like 80 without root privileges. This tutorial doesn't go into managing permissions, but we'll show you how to change the port and assume you'll take care of the permissions safely.

For quick, temporary servers on your local machine, you can use the -port command line option:

Windows
C:\Caddy\caddy.exe -port 8080
macOS and Linux
caddy -port 8080
